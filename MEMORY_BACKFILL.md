## Foundational Idea 3: SDS as Operating System

**When:** Phase 6 (February 2026)  
**What:** Created Strategic Decision Engine (SDS) to govern all work prioritization  
**Why:** Reactive yes/no responses to opportunities were causing cognitive overload. Systematic scoring needed.

**SDS Formula:**
```
SDS = (Revenue × 2) + (Alignment × 2) + (Leverage × 1.5) + (TimeFit × 1) - (CognitiveLoad × 1)
```

**Key Implications:**
- All opportunities scored before decision
- Clear thresholds: YES ≥ 25, PROBABLY 18-24, LATER 12-17, NO < 12
- Autonomous operation without asking

## Foundational Idea 4: Content as Leverage

**When:** Phase 4-5 (February 2026)  
**What:** Shifted from "content ideas" to "content as asset"  
**Why:** Content creation was ad-hoc. Systematic approach needed for compounding leverage.

**Key Multipliers:**
- Framework = 10x (reused forever)
- Tool/Prototype = 5x (users return)
- Research Paper = 3x (cited forever)
- Case Study = 2x (sales enablement)
- Speaking = 1x (one-time reach)
- Blog Post = 0.5x (time > value)

**Key Implications:**
- Content prioritized by multiplier, not novelty
- Framework creation preferred over blog posts
- Every content piece must have leverage path
